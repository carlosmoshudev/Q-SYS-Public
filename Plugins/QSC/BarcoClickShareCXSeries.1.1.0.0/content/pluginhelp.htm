Barco ClickShare CX Series

Use this plugin to connect to, control, and view the status of a Barco ClickShare CX Series Wireless Conferencing System.

Note: See the Barco website for product information and documentation.

Compatibility

The plugin was developed and tested with Q-SYS Designer version 8.3.3 and Barco firmware version 2.5.1.2. The plugin supports the CX-20, CX-30, and CX-50 models.

Prerequisites

Before using the plugin, you must enable the "Remote control via API" option from the Barco device's web interface:

  1. In a browser, use the Barco device's IP address to navigate to the web configuration interface.
  2. Specify your credentials to log in.
  3. From the left menu, go to Wi-Fi & Network > Services.
  4. Under the ClickShare API section, select to enable the "Remote control via API" option.

Note: The Remote control API credentials (used by the plugin) are the same as those for the web configuration interface. Go to Security > Passwords to set this password.

Connecting to the Device

To begin using the plugin, drag it into the schematic and configure its Properties. Then, press F5 to save your design to the Core and run it.

In the plugin's Setup tab:

  1. Type the IP Address of the device.
  2. Specify your API Username and Password credentials for the device, and then press Enter.

The plugin will automatically attempt a connection. If you see "OK" status, you are successfully connected to the device. If you see a "Fault" error, check to make sure you entered the correct parameters.

Properties

Is Managed

Select 'Yes' to add the plugin to your Inventory list, which makes the plugin's Status available for monitoring in Core Manager and Enterprise Manager. When enabled, you can then specify a Name and Location for the plugin, as with any other Inventory item.

For more information, refer to the Plugins topic in the Q-SYS Help.

Paired Buttons

Specify the number of buttons you have paired to the Base Unit, from 1 to 32 (default is 1).

Note: If you specify more buttons than are returned by the device, the controls appear in the plugin but display a fault or error message.

Show Debug

Select 'Yes' to show the Debug Output window. For details, see the Debug Output topic in the Q-SYS Help.

Controls

Setup

IP Address

The IP address of the Barco device.

Username

This is the same username as for the device's web configurator.

Password

This is the same password as for the device's web configurator.

Status

Displays the current connection status:

  • OK: Plugin is connected to the unit
  • FAULT: Check the credentials to connect to the device.
  • MISSING: No device is found at the specified IP address.

Sharing

Denotes whether any buttons are currently sharing to the base unit.

Reboot System

Click to send a reboot command to the base unit. During the reboot cycle, the plugin loses connection to the device but will automatically re-connect when the device is back online.

Base Model

Displays the model of the connected base unit.

Serial Number

Displays the serial number of the connected base unit.

Buttons

Connected

LED indicates whether the paired button is connected to the base unit.

Serial Number

Displays the serial number of the paired button.

MAC Address

Displays the MAC address of the paired button.

Firmware

Displays the firmware of the paired button.

Status

Displays the status of the paired button - for example, "Ok".

Video

Video Output > Connected

LED indicates if there is a video output connected to the base unit.

Video Output > Preferred Resolution

Displays the preferred resolution of the connected display.

Video Input > Connected

Supported by CX-50 only. LED indicates if there is a video input connected to the base unit.

Video Input > Source Name

Supported by CX-50 only. Displays the source name of the connected input, as set in the web configurator in the Display & Audio > Inputs section.
