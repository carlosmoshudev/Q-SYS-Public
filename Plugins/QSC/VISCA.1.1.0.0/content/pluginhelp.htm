VISCA Control Server

Use this plugin to allow third party camera control interfaces (supporting the VISCA over IP command protocol) to connect to a Q-SYS Core processor and control native Q-SYS cameras using TCP, UDP or Serial.

Compatibility

Plugin version 1.1 was designed and tested with Q-SYS Designer Software version 9.10.2 using the PTZOptics camera control application for Windows, the Skaarhoj PTZ Fly with the VISCA device core, an AIDA CCU-IP (for serial control testing), a Q-SYS Core 110f processor, and a Q-SYS NC-12x72 camera.

Requirements

The plugin uses Named Components to identify and connect to the various cameras in a design, so in Q-SYS Designer Software versions 9.5 and greater, the Script Access property must be set to set to "All" or "Script" to allow cameras within the design to show up as choices in the combo boxes on the Setup page.

Configuration

To configure the VISCA Control Server plugin:

  1. Ensure that there is a Q-SYS camera connected to the system to be controlled.
  2. If using the TCP control mode, determine the control port used for VISCA over IP commands by the third party camera controller. This value differs across manufacturers and is often hard-coded into their implementations; for example, PTZOptics uses port 5678 for their Windows-based control application.
  3. If using the UDP control mode, ensure that your client device is set to use the standard VISCA over IP control method via UDP. This should be done via port 52381.

    Note: When using UDP mode, do not use multiple instances of the VISCA plugin.

  4. If using the Serial control mode, note the serial settings that the third party controller requires to connect (Baud Rate, Data Bits and Parity).
  5. Load the plugin and run the design. Once initialized, if using the TCP control mode, enter the appropriate port number for the server to listen on. If using the Serial control mode, enter the Baud Rate, Stop Bits, Data Bits and Parity. Map your cameras using the combo boxes on the setup page.
  6. Once a camera is mapped, be sure to select that camera on the Controls page.
  7. Execute a command from the third party controller and look for status in the plugin to change to OK when a successful connection is made.
    • Note that if you are using the TCP control mode, you will likely need to enter one of the IP addresses of the Core somewhere within the third party controller during setup.
    • Check that the commands are passed along to the selected camera on the Controls page by looking directly at the output of that camera or at the corresponding controls on the status block.

Inputs and Outputs

Input Pins

Camera Select

These are toggle button inputs that correspond to the 6 camera selection buttons on the Controls page. These buttons determine which of the 6 cameras in the design, configured via the 6 drop down combo boxes on the Setup page, will be controlled by commands received from the third party control device.

Port

When the plugin is configured in TCP control mode, this pin will modify the network port that the plugin listens on for commands from third party devices. Note that changing the port will terminate all open and active connections, and the plugin will start listening on the new port. The plugin supports multiple, discrete TCP client connections in a single instance.

Note: To listen on multiple ports, load multiple instances of the plugin. However, avoid using multiple instances set to the same listen port.

Note: When the plugin is configured in UDP control mode, it uses the standard VISCA UDP port: 52381.

Baud Rate, Data Bits, and Parity

When the plugin is configured in Serial control mode, these pins will modify settings for the serial connection.

Output Pins

PTZ, Focus, and Zoom

These are LED indicator outputs that pulse when the associated control is active on the third party control device. These controls pass via Named Components to the selected camera within the plugin, so there is no need to wire these pins to Q-SYS camera status blocks for operation; however, these pins are still available should you need them. Note that the Center LED will illuminate when the joystick is not in motion.

Advanced

These outputs correspond to the various advanced camera controls found on the Advanced page of the plugin. Like the PTZ, Focus, and Zoom controls, these controls also pass to the selected camera within the plugin via Named Components. There is no need to wire these directly to camera status blocks for the plugin to operate, but should you need them, they are available.

Properties

Global Properties (All Devices)

Is Managed

Select 'Yes' to add the plugin to your Inventory list, which makes the plugin's Status available for monitoring in Core Manager and Enterprise Manager. When enabled, you can then specify a Name and Location for the plugin, as with any other Inventory item.

For more information, refer to the Plugins topic in the Q-SYS Help.

Plugin Mode

Select the method for the control server to use to listen to commands. Refer to the Configuration section for details and instructions on each mode. The three modes for control are TCP, UDP and Serial.

Controls

Setup

Port

When the plugin is configured in TCP control mode, this pin will modify the network port that the plugin listens on for commands from third party devices. Note that changing the port will terminate all open and active connections, and the plugin will start listening on the new port. To listen on multiple ports, load multiple instances of the plugin.

Baud Rate, Data Bits, and Parity

When the plugin is configured in Serial control mode, these pins will modify settings for the serial connection.

Camera Selection Combo Boxes

These combo boxes let you map a camera, 1 through 6, to a camera present and connected within the design. This is how the plugin knows which camera to control when a camera button is selected on the Controls page. Note that the plugin uses Named Components to identify and connect to the various cameras in a design, so in Q-SYS Designer versions 9.5 and greater, Script Access must be set to set to "All" or "Script" to allow this.

Connection / Configuration Status

Indicates the current status of the connection. If you see "Missing", check that you have entered the correct IP address of the Core for TCP connections from controller devices. This status box will also indicate a fault when a camera is not selected/configured in the design, when a port is not set for listening in TCP mode, and when the serial port is not fully configured in serial mode.

Controls

Camera Selection Buttons

These are toggle button inputs that correspond to the 6 camera selection buttons on the Controls page. These buttons determine which of the 6 cameras in the design, configured via the 6 drop down combo boxes on the Setup page, will be controlled by commands received from the third party control device.

PTZ, Focus, and Zoom

These are LED indicator outputs that pulse when the associated control is active on the third party control device. These controls pass via Named Components to the selected camera within the plugin, so there is no need to wire these pins to Q-SYS camera status blocks for operation; however, these pins are still available should you need them. Note that the Center LED will illuminate when the joystick is not in motion.

Advanced

White Balance Mode

This corresponds to the white balance mode of the camera, and supports Auto, Manual, and OnePush.

White Balance Red/Blue

When White Balance is in Manual Mode, these controls correspond to the manual red/blue gain of the camera's white balance. When set to Auto, the blue and red limits are not supported.

Exposure Mode

This corresponds to the exposure mode of the camera, and supports Auto and Manual.

Shutter

This corresponds to the shutter setting of the camera. Q-SYS cameras support all of the shutter settings present in the VISCA protocol.

Aperture/Iris

This corresponds to the aperture setting of the camera. Q-SYS cameras support all of the aperture settings present in the VISCA protocol.

Backlight Compensation

This LED indicates whether or not the backlight compensation has been turned on or off by the connected third party control device.
