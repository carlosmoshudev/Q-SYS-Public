Command Send And Receive Utility

The Command Send and Receive Utility plugin enables non-scripting users to send data over a socket and interpret responses. It simplifies integration, enhances automation, and eliminates the need for scripting.

Compatibility

This plugin was developed and tested with Q-SYS Designer Software version 9.10. It establishes a connection with the TCP, HTTP, UDP, SSH, and Serial protocols based on the control type of the endpoint. To begin using the plugin, drag it into the schematic from the Plugins > Asset Manager > Utility folder.

Properties

Control Type

Select a control type (TCP, HTTP, UDP, SSH, and Serial) from the Properties panel. If the control type is HTTP, then an additional field called "Authentication Required" (with a "Yes" or "No" choice) will be added.

Send and Receive String Count

The Send and Receive string text fields can be adjusted within a range of 1 to 64 from the Properties panel during design time. Depending on the Send String Count, controls will be drawn in the Send tab. Based on the Receive String Count, controls will be drawn in the Receive tab.

Buffer Caching

When configuring the Buffer Caching functionality in the plugin, you have the option to choose either "Time" or "Count". If you select "Time", the Buffer Refresh interval can be set from 1 to 1800 seconds. This setting ensures that the responses from the endpoints are displayed in the Buffer Received Data panel for a specified duration based on the Buffer Refresh interval.

Time:

If you select "Time", the Buffer Refresh interval can be set from 1 to 1800 seconds. This setting ensures that the responses from the endpoints are displayed in the Buffer Received Data panel for a specified duration based on the Buffer Refresh interval.


Count:

If you select "Count", an additional field called "Message Cache Amount" will be added. You can select a value from 1 to 100 for this setting. It ensures that the responses data from the endpoints is displayed in the Buffer Received Data panel for a specified amount set in "Message Cache Amount".

Show Debug

Select 'Yes' to show the Debug Output window. For details, see the Debug Output topic in the Q-SYS Help.

Debug Print

Set the data printed to the Show Debug window.

  • None: (default) None of the below will be posted to the log debug output.
  • Tx/Rx: Transmission and Received Data messages will be shown.
  • Tx: Transmission Data messages will be shown.
  • Rx: Received Data messages will be shown.
  • Function Calls: Each function within the plugin will post a message when it is called.
  • All: All of the above messages.

Script Access Properties

Code Name

Displays the currently assigned name for control access. You can use the auto-assigned name or customize it. Q-SYS will automatically check all Code Names in the design to ensure the name is unique.

Script Access

Defines whether the component will be accessible by script and/or externally, or not at all. Choices include All, External, None (default), and Script.

Tip: Use Script Programmer Mode to quickly view the Script Access setting directly on the component in the design schematic without the need to disconnect from the Q-SYS Core processor.

Note: Enable script level access for both the HID Conferencing and Bin Loop components.

Controls > Connection Setup Tab

TCP

IPAddress

By default, it will be empty during runtime.

Port

By default, it will be set to 80 during runtime. However, you can modify this value between 1 and 65535 according to your requirements.

EOL (End of Line)

By default, it will be set to “None”. However, you can select a preferred EOL from the dropdown.

EOLCustom

This option will only be enabled when you select “Custom” as the EOL from the dropdown; otherwise, it will remain disabled.

Status

  • When no IPAddress is entered, the status will be displayed as “MISSING - No Valid IP Address”.

  • When a valid IPAddress or hostname is entered along with the correct port, the status will be displayed as “OK - Connected”.

  • If the port does not match with the physical device, the status will be displayed as "FAULT - Socket Error".

  • When the TCP socket is closed, the status will be displayed as "MISSING - Socket Closed".

  • If the TCP socket encounters an error, the status will be displayed as "FAULT - Socket Error".

HTTP

When Authentication is set to No (by default) from the properties panel:

When Authentication is set to Yes from the properties panel:

URL

Enter a valid URL. If the URL necessitates a port, it should be included in the URL itself (e.g., http://<ip-address> or <hostname>:port).

Username

The username is mandatory for authentication-based connections and must adhere to the configuration requirements of the endpoint.

Password

A password is necessary for authentication-based connections and must be a valid password according to the endpoint configuration.

Auth

Authentication is required for endpoints that demand it. By default, Auth will be set to "any", and you can select from the dropdown menu based on your requirements.

Headers

You should provide headers in a valid format in the text field (e.g., ["Content-Type"] = "application/json"). If additional headers are required for the request, they can be appended in comma-separated key-value pairs.

Status

  • When the URL field is empty, the status will be displayed as "MISSING - No URL".

  • When the HTTP connection is established with the endpoint, the status will be displayed as "OK".

  • If the request to the endpoint results in a successful 200 code, the status will be displayed as "OK - 200".

  • If the response returns a 400 code, indicating a malformed request, the status will be displayed as "FAULT - Malformed Request".

  • A 401 code in the response will result in the status being displayed as "FAULT - Unauthorized".

  • A 403 code in the response will result in the status being displayed as "FAULT - Forbidden".

  • If the response returns a 404 code, indicating a resource not found, the status will be displayed as "FAULT - Resource not Found".

  • If the response returns a 500 code, indicating an internal server error, the status will be displayed as "FAULT - Internal Server Error".

  • If the response returns a 503 code, indicating a service unavailable, the status will be displayed as "FAULT - Service Unavailable".

UDP

IPAddress

By default, it will be empty during runtime.

Port

By default, it will be set to 80 during runtime. However, you can modify this value between 1 and 65535 according to your requirements.

Heart Beat Time

By default, it will be set to 3 during runtime. However, you can modify this value between 3 and 300 according to your requirements. This will help to monitor the status of the endpoints. If the endpoint fails to respond to the heartbeat, it indicates a potential issue that needs attention.

Status

  • When no IPAddress is entered, the status will be displayed as "MISSING - Invalid IP Address".

  • If the UDP socket successfully connects, the status will indicate "OK".

  • If there's an error with the UDP socket connection, resulting in the inability to open the UDP socket, the status will be displayed as "FAULT - Socket Error: Unable to open UDP socket".

SSH

IPAddress

By default, it will be empty during runtime.

Port

By default, it will be set to 22 during runtime. However, you can adjust this value between 1 and 65535 based on your requirements.

Username

A username is required for authentication-based connections and must adhere to the configuration requirements of the endpoint.

Password

A password is required for authentication-based connections and must be a valid password according to the endpoint configuration.

EOL (End of Line)

By default, it will be set to "None". However, you can select your required EOL from the dropdown.

EOLCustom

This option will only be enabled when you select “Custom” as the EOL from the dropdown; otherwise, it will remain disabled.

Status

  • When no IPAddress is entered, the status will be displayed as "MISSING - Invalid IP Address".

  • If the SSH socket successfully connects with a valid IP address, port, username, and password, the status will be displayed as "OK - Connected".

  • If the SSH socket connection is closed, resulting in a socket closure, the status will be displayed as "MISSING - Socket Closed".

  • If there's an error with the SSH socket connection, the status will be displayed as "FAULT - Socket Error".

  • If the SSH socket connection times out, resulting in a socket timeout, the status will be displayed as "MISSING - Socket Timeout".

  • If the SSH socket login fails, indicating incorrect credentials, the status will be displayed as "MISSING - Check credentials".

Serial

BaudRate

You can select the required baud rate from the combobox based on your needs. By default, it will be set to 9600.

DataBits

You can select the required number of data bits from the combobox based on your needs. By default, it will be set to 8.

Parity

You can select the required parity from the combobox based on your needs. By default, it will be set to None.

DataBits

By default, it will be set to 80 during runtime. However, you can modify this value between 1 and 65535 according to your requirements.

StopBits

You can select the required number of stop bits from the combobox based on your needs. By default, it will be set to 1.

EOL (End of Line)

By default, it will be set to “None”. However, you can select your preferred EOL from the dropdown.

EOLCustom

This option will only be enabled if you select “Custom” as the EOL from the dropdown; otherwise, it will remain disabled.

Heart Beat Time

By default, it will be set to 3 during runtime. However, you can modify this value between 3 and 300 according to your requirements. This will help to monitor the status of the endpoints. If the endpoint fails to respond to the heartbeat, it indicates a potential issue that needs attention.

Status

  • When all settings are correct with the physical device and it is connected, the status will be displayed as "OK - Connected".

  • If a serial port is closed, resulting in a serial port closure, the status will be displayed as "MISSING - Serial port Closed".

  • If there's an error with the serial port connection, the status will be displayed as "FAULT - Serial port error".

  • If the serial port connection times out, resulting in a serial port timeout, the status will be displayed as "MISSING - Serial port timeout".

Controls > Send Tab

TCP, UDP, SSH and Serial

During runtime, after the selected control type socket (TCP or UDP or SSH or Serial) is properly connected to the endpoint, you can send plain text or regex control commands to the endpoint from the Send page by entering commands in the text field and triggering the button.

HTTP

During runtime, you can send plain text or regex control commands to the endpoint from the "Send" page by entering commands in the Command text field. If any data is required to pass based on the endpoint's API (using GET or POST requests), you can add it in the Data field before triggering the button. Before pressing the trigger button, select GET, POST, PUT, PATCH, or DELETE from the combo box. GET will be selected by default.

Controls > Receive Tab

Once the string/command has been sent to the endpoint from the Send page, the device will provide a response. If the text/string matches the response in the Receive page text field, then the LED will go high for 3 seconds before returning to low. The LED will remain low if the received data does not match. During runtime, the LED will be low by default.

Controls > Buffer Tab

If Buffer Caching is set to "Time", the Buffer Refresh interval can be set from 1 to 1800 seconds. This setting ensures that the responses from the endpoints are displayed in the Buffer Received Data panel for a specified duration based on the Buffer Refresh interval.

If Buffer Caching is set to "Count", an additional field called "Message Cache Amount" will be added. You can select a value from 1 to 100 for this setting. It ensures that the responses data from the endpoints is displayed in the Buffer Received Data panel for a specified amount set in "Message Cache Amount".
