1. Prerequisites

Before configuring the plugin, ensure that:

  • The Absenicon display is connected to a network and is assigned a valid IP address which the Q-Sys Core can communicate with.
  • The assigned IP address of the Absenicon display will not change, either by assigning a static address or reserving an address for the display MAC address.

2. Configuration

  1. Drag the plugin into the schematic design.
  2. Select the plugin component and configure the Properties as desired (see below for Properties descriptions).
  3. Save the design file and either:
    1. Push the design to the Q-Sys Core, or
    2. Emulate the design
  4. In the schematic design, double click the plugin component to access the plugin controls.
  5. On the Setup page, enter the Absenicon display's IP address into the IP Address text control (see below for Controls descriptions).
  6. The plugin will then attempt to connect to the display. Upon successful connection, the Status control will show "OK".

6. Notes

6.1. Brightness is set when first connected

The display control protocol does not allow for the brightness level to be queried from the display. This can result in mismatched levels between the plugin value and the actual brightness of the display. To limit this issue as much as possible, the brightness level set in the plugin is sent to the display upon initial connection. The display brightness level will then match the plugin level and should stay in sync with each other, so long as the plugin is the only method of control over the brightness.

6.2. Returning to a non-split screen state (Ultrawide model only)

Once a split screen mode has been set on the display, the split screen control is only able to change the type of split screen; the "None" option is feedback only and does not set the screen into a non-split state. In order to get out of split screen, send an Input Select command, either by selecting one of the HDMI buttons or selecting an option from the Input Selection combobox.
