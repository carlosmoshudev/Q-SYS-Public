FIFO Queue

The first in, first out (FIFO) Queue plugin is a tool that enables you to control when incoming lines of text are sent to a downstream scripting or control component. For example, some devices might not be able to handle multiple commands at once and require acknowledgment between commands. With this plugin, you can control the timing – either manually or automatically – of sending commands downstream.

Compatibility

The plugin was developed and tested with Q-SYS Designer Software version 9.0 and also tested to work with version 8.4.

Configuration Overview

To begin using the plugin, drag it into the schematic.

  1. By default, two control pins are exposed:

    • Queue Input: Wire this pin to a text edit control for adding data to the queue – for example, Custom Controls.

    • Queue Output: Wire this pin to a text display control for sending data over a socket – for example, Custom Controls.

  2. Press F5 to save your design to the Core and run it.

  3. Double-click the plugin to open its control panel.

  4. Configure the FIFO Queue controls. See Controls.

Properties

Show Debug

Select 'Yes' to show the Debug Output window. For details, see the Debug Output topic in the Q-SYS Help.

Controls

Sending Configuration

Send Next

A trigger button that sends the next line of data over the Queue Output pin. The line is then removed from the Queue display.

Sending

LED is on / glows red when a line is being written to the Queue Output pin.

Auto Send

Toggle this on to automatically send the next line of data over the Queue Output pin at the Queue Time interval.

Queue Time

Used in conjunction with the Auto Send control, this sets how frequently the next line of data in the queue is sent over the Queue Output pin. This control is disabled when Auto Send is disabled.

Queue

Clear

Click to clear all data currently in the queue.

Queue

Displays the data currently in the queue, with the first at the top.

Example

In this example, a Custom Controls text box is wired to the FIFO Queue input pin to provide a way to input data - in this case, a few lines of text. Another Custom Controls component is configured as a text display and wired to the FIFO Queue output pin to show what data is currently being sent. The FIFO Queue component is configured to automatically send the next line of text every 10 seconds.
