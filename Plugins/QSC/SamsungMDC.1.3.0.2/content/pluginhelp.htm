Samsung Commercial Display (MDC)

Use this plugin to connect to and control any Samsung Commercial Display that supports the MDC communication protocol. The plugin can connect over an Ethernet (or WiFi) network or by serial via RS-232. You can control the display's power, input mode, and volume. The plugin supports one display or can broadcast control commands to a serial daisy chain.

Note: See the Samsung website for product information and documentation.

Compatibility

The plugin was developed and tested with Q-SYS version 8.4 and a Samsung model QM43R running firmware version T-KTM2ELAKUC-2081.1.

Known Issue: Some QB75 model firmware versions might cause a large amount of data to be sent by the TV at start-up. In such cases, it can take over 30 seconds for the plugin to show valid data from the TV following a power-on. If this issue is diagnosed, we recommend customers report it the TV manufacturer.

Getting Started

Follow Samsung guidelines on cables and connections to use with your display. Some Samsung displays support WiFi connections, which can be used in place of Ethernet as long as the Q-SYS Core and Samsung display can communicate over the network.

Note: WiFi connected displays do not support power-on functionality from the plugin.

1. Connect the Display

Ethernet – Single Display

Directly connect your Samsung display to the network, and connect over Ethernet or WiFi by IP address.

  1. Connect an Ethernet cable from the display's Ethernet port to the network switch. (Or, connect the display to your network using WiFi).
  2. Connect the Q-SYS Core processor to the same network. Configure the display and the Q-SYS Core to use the same network segment.

Note: You can alternatively connect the display directly to one of the Q-SYS Core's LAN ports.

Ethernet – Multiple Display Serial Daisy Chain

Connect the first display to the Q-SYS network via Ethernet and subsequent displays via daisy chained serial.

  1. Connect an Ethernet cable from the display's Ethernet port to the network switch.
  2. Connect the Q-SYS Core processor to the same network. Configure the display and the Q-SYS Core to use the same network segment.
  3. Use an 8mm TRS cable to connect the first display's "RS232 OUT" port to the "RS232 IN" port on the second display.
  4. Continue to connect additional displays as in step 3.

Note: In this configuration, the display with the Ethernet connection must be powered on for commands to reach the serial chain.

Serial – Single Display

Directly connect your Samsung display's RS-232 port to an RS-232 port on your Q-SYS Core processor:

  1. Acquire an RS-232 converter cable with a female DB9 connector on one end and male TRS 8mm on the other. Confirm the pinout matches Samsung's requirements.
  2. Connect the TRS 8mm end to the Samsung Display's port labeled "RS232 IN".
  3. Connect the DB9 end to the Q-SYS Core's RS-232 port.

Serial – Multiple Display Daisy Chain

To connect a serial daisy chain of Samsung displays together:

  1. Connect the Q-SYS Core to the first display using an RS-232 converter cable with a female DB9 connector on one end and male TRS 8mm on the other. Confirm the pinout matches Samsung's requirements.
  2. Use an 8mm TRS cable to connect the first display's "RS232 OUT" port to the "RS232 IN" port on the second display.
  3. Continue to connect additional displays as in step 2.

2. Set up the Display

Power on each display and run the on-screen setup. Consult the Samsung documentation for your display on how to connect cables and set up connections within the display's software.

  • Device ID – Each display in a serial daisy chain must be assigned a unique Device ID.
  • PC Connection Cable – Set to "RS232C" or "Ethernet" based on how you connected the display to the Q-SYS Core.

3. Configure the Plugin

  1. Drag the plugin into your schematic.
  2. Select the plugin to configure its Properties, including the Connection Type (Ethernet or Serial) and Poll Interval for plugin status (from 1 to 60 seconds). For details, see Properties.
  3. For serial connections only, drag out a Serial Port component from the Core's inventory tree and wire it to the Serial input pin of the plugin.

  4. Press F5 to save the design to the Core and run it.
  5. Power on the Samsung display.
  6. Double-click the plugin to open the control panel.
  7. In the Setup tab, configure the connection parameters. Set the Display ID to the display you wish to control, from 0 to 253.
    • For Serial daisy chains, you can address any display with the chain. The display to which the Core is connected sends commands to the other displays on the chain until the correct display receives the message.
    • For Ethernet connections, specify the IP Address and Port (default is 1515 for the MDC protocol).

      Note: For the power-on command to work, the MAC address must populate in the Status section. The display must be powered on before connecting with the plugin so that the MAC address can be acquired. For Ethernet connections with multiple displays connected via serial daisy chain, the power-on command only functions if the Ethernet-connected display is targeted with the Display ID.

  8. Click the Control tab. You can now control the display.

Properties

Is Managed

Select 'Yes' to add the plugin to your Inventory list, which makes the plugin's Status available for monitoring in Core Manager and Enterprise Manager. When enabled, you can then specify a Name and Location for the plugin, as with any other Inventory item.

For more information, refer to the Plugins topic in the Q-SYS Help.

Connection Type

Select how the display is connected to the Q-SYS Core, either Ethernet (default) or Serial.

Poll Interval

Specify how often the plugin polls the display for updated status, from 1 to 60 seconds (default is 3).

Show Debug

Select 'Yes' to show the Debug Output window. For details, see the Debug Output topic in the Q-SYS Help.

Controls

Setup

IP Address

In Ethernet mode, specify the IP address of the display.

Port

In Ethernet mode, specify the communication port for the MDC protocol. The default port is 1515. In most cases, leave as-is.

Display ID

Required in all connection modes. Set the ID to match the display to control.

Reboot

Click to reboot all connected displays. (This is a Broadcast Mode command.)

Status

Displays the current connection status of the display with the targeted Display ID:

  • OK: Plugin is connected to the device.
  • FAULT: Indicates a connection problem with the device, such as a closed socket due to device failure.
  • MISSING: No device is found at the specified IP address.

Device Name, Model Name, Serial Number, Software Version, MAC Address

These read-only values are obtained from the display once the plugin successfully connects.

Control

Broadcast Mode

Turn on to send broadcast commands instead of individually addressed commands. Individual commands will go to the display set with Display ID. Broadcast commands will be sent to all displays connected in a serial daisy chain.

Power > On

Click to turn on the display. Supports Broadcast Mode.

  • Serial connections: No restrictions.
  • Ethernet connections: Plugin must connect to the display and acquire the MAC address before this function will work. Note that the MAC address is retained after connecting for the first time.
  • WiFi connections: Power-on is not supported.

Power > Off

Click tot turn off the display. Supports Broadcast Mode.

Power > Status

LED indicates whether the connected display is powered on.

Current Input

The name of the current input in use. Supports a single target display.

Input Buttons

There are 27 defined input modes that have a discrete button to set the display to that mode. An LED indicates which input is active. Supports Broadcast Mode. Note that available inputs vary by Samsung display model.

Volume

Set the volume in 1 value increments from 0 to 100. Supports Broadcast Mode.

Mute

Click to toggle display muting. Supports Broadcast Mode.
